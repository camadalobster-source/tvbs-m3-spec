404 / 500 錯誤頁面

404 Not Found
何時出現
URL 路徑不存在、拼字錯誤、文章永久刪除/下架、時效性專區移除
Server 找不到對應路由 → 直接回 HTTP 404
核心元件
搜尋框（P0）+ 回首頁按鈕（P1）
部署方式
應用程式內部渲染（維持原 URL）
Header
Sticky，含完整導航
標題文案
「很抱歉，頁面無法使用或被移除，試試別的吧?」
500 Internal Server Error
何時出現
核心 API 完全無回應、系統崩潰、SSR 整頁渲染失敗
Server 被動產生，前端不主動 redirect
核心元件
回首頁按鈕 + 官方社群連結（FB / Line / YouTube）
部署方式
代理伺服器層級，純靜態 HTML/CSS
Header
靜態 Logo（建議不依賴 API）
標題文案
「系統忙碌中，請稍候片刻重新嘗試」
開發注意事項
  • 1404 必須在原 URL 回傳 HTTP 404，禁止 redirect 到 /404 路徑
  • 2500 零依賴：CSS inline / Base64 圖片，不經 Next.js 渲染；Nginx 設定 error_page 500 502 504
  • 3前端禁止主動 redirect 至 404 / 500，錯誤頁由 Server 被動產生
  • 4無效分頁參數（如 ?page=9999）用 302 導回第一頁，禁止 301
  • 5GA4 追蹤：統一 Page Title「404 找不到網頁 | TVBS 新聞網」，不加 UTM / from 參數
  • 6Slug 變更時 CMS 須自動建立 301 Redirect 並同步更新 canonical
  • 7局部元件失效（廣告、非核心 API）用 Error Boundary 隱藏，核心內容在就回 200

Cookie Banner 隱私權置底橫幅

✓ 點擊「我同意」
橫幅立即隱藏（display: none）
寫入 Cookie
Key：tvbscookie
Value：{ status: 1, timestamp: ISO-8601 }
Domain：tvbs.com.tw
Expires：寫入起算 365 天
✕ 點擊「X」關閉
橫幅立即隱藏（display: none）
不寫入任何 Cookie 或 localStorage
下次重新整理或進站，橫幅再次顯示
顯示邏輯
條件行為
Cookie 不存在顯示橫幅
Cookie 存在，timestamp 未逾 365 天不顯示
Cookie 存在，timestamp 已逾 365 天視為過期，重新顯示橫幅
使用者點 X（無 Cookie 寫入）下次進站重新顯示
注意事項
  • 1position: fixed; bottom: 0; width: 100%
  • 2z-index 建議 999，低於蓋板廣告（Overlay Ad）或 Modal
  • 3「隱私權政策」連結開新分頁 → account.tvbs.com.tw/policy/privacy
  • 4橫幅不應自動無互動消失；換裝置 / 瀏覽器 / 清資料均視為新訪客
  • 5MVP 不強制無障礙（無須 aria-label / 鍵盤完整操作）

🔍 標籤查詢 + 黑名單過濾機制

第一層：標籤聯集查詢 (OR)
固定標籤：氣象、天氣、颱風、低溫、高溫、豪雨、豪大雨
依發布時間 DESC 排序
第二層：後端 ID 黑名單排除
查詢加 WHERE article_id NOT IN (Blacklist)
Blacklist 為空時跳過 NOT IN 條件
為什麼需要黑名單？
標籤受 SEO 策略、編輯習慣、AI 自動生成影響，會出現語意污染。例如「颱風天必吃泡麵」命中 #颱風 但內容是超市促銷；「颱風救災預算攻防」命中 #颱風 但內容是政治議題。單靠標籤位置、數量等規則無法穩定排除。
MVP 建議作法：Config 檔（JSON / YAML）存放 ID 清單，修改後透過一般部署流程生效，不需額外後台。不開 CMS 管理介面。未來若需多人即時維護，再升級至 DB 表。

搜尋結果頁 — 區塊與互動

搜尋框
  • 輸入值先 .trim()
  • trim 後空字串 → 不搜尋
  • Enter 或點搜尋 icon 觸發
  • 搜尋中顯示 Spinner +「搜尋中」
歷史搜尋
  • localStorage 存 5 組 max
  • Key: tvbs_search_history
  • 標籤可單則 X 刪除
  • 10 非空白字元截斷 + ...
  • Desktop hover 顯示全文
搜尋摘要
  • 搜尋：{關鍵字} 結果共 {N} 筆
  • N = API total
  • 關鍵字截斷同歷史搜尋規則
結果列表
  • 初始 16 則 + 載入更多 16 則
  • 排序：updated_at DESC
  • MVP 僅搜尋文章
  • 影音新聞加 Play overlay
  • 點擊另開新分頁

📦 歷史搜尋 localStorage 淘汰邏輯

Value 結構：[{ "keyword": "關鍵字", "timestamp": 1234567890 }]，依 timestamp DESC
新關鍵字：已有 5 筆 → 刪最舊一筆再新增；未滿直接新增
重複關鍵字：僅更新 timestamp，不新增筆數，不觸發淘汰
手動刪除：可少於 5 筆，下次新關鍵字寫入再補滿
儲存內容：存 trim 後完整字串，不因標籤顯示截斷而縮短

搜尋結果頁 — 狀態與降級處理

SEO

Title：搜尋結果：{關鍵字} - TVBS新聞網
Robots：noindex（不被搜尋引擎索引） | 不需要 OG Tags、不需要 JSON-LD、不設 Canonical

應用程式層狀態一覽
狀態觸發條件畫面行為
搜尋服務失敗OpenSearch 5xx / fetch timeout顯示「搜尋服務暫時無法使用」；搜尋框仍可操作
結果為空API 正常，結果 0 筆「找不到相關結果」；歷史搜尋仍顯示
搜尋中等待 API 回應Spinner +「搜尋中」；非錯誤狀態
載入更多失敗分頁 API 失敗 / timeout按鈕變「再試一次」；已載入結果不動
空白輸入.trim() 後為空字串不執行搜尋；維持當前狀態
局部失效不影響整頁 ｜ 可重試的錯誤保留使用者的關鍵字
